## Context

現在のアプリは、ダークテーマのUIと単一のRSSフィード（NHKニュース）からのニュース取得のみに対応しています。ユーザーから以下の改善要望が出ています：
1. **UI改善**: ホワイト基調でポップで明るいデザイン、記事タイトルの日本語表示改善
2. **ニュース機能**: 複数カテゴリからのニュース取得、段階的な件数制限（主要10件、カテゴリ別5件）

現在の制約：
- フロントエンドはプレーンなHTML/CSS/JavaScriptで実装
- バックエンドはPython（`fetch_news.py`）で実装
- 出力は `data/news.json` で固定
- Gemini APIを使用してニュース翻訳・要約を生成

## Goals / Non-Goals

**Goals:**
- ホワイト基調でポップで明るいUIテーマを実装し、ユーザーの視認性を向上させる
- 記事タイトル表示を最適化し、長いテキストでも日本語が読みやすい状態を実現
- NHKニュースの複数カテゴリ（主要ニュース、地域別ニュース等）から段階的にニュースを取得する仕組みを構築
- ニュースデータ構造をカテゴリ別に整理し、フロントエンドでカテゴリ別表示が容易にできるようにする

**Non-Goals:**
- ユーザー認証・ログイン機能
- ローカライゼーション（日本語以外の言語対応）
- 既存の英語音声再生・ハイライト機能の変更（player-interface の既存機能は維持）
- 新規外部APIの追加（Gemini API の使用は継続）

## Decisions

### 1. UIテーマの実装方式：CSS カスタムプロパティ（変数）を使用
**決定**: `style.css` にホワイト基調の色変数セットを定義し、既存のダークテーマ変数を置き換える形で実装

**根拠**:
- 既存HTMLスクリプト構造に最小限の変更で対応可能
- 再利用可能で保守しやすい
- 将来のテーマ切り替え機能拡張にも対応可能

**検討した代替案**:
- HTMLファイル内のインラインスタイル変更 → 保守困難、スケーラビリティに欠ける
- 新規CSSファイルの作成 → 不要な複雑性を増加

### 2. 記事タイトル表示の最適化：`word-break` 、行高さ、テキスト配置の調整
**決定**: CSS で以下を適用
- `word-break: break-word` で適切な折り返しを実現
- `line-height: 1.6` 以上で日本語の可読性を向上
- タイトル表示エリアの幅制限と、複数行対応レイアウト

**根拠**:
- CSSのみの変更で、JavaScriptコードの追加が不要
- ブラウザネイティブの文字折り返しが安定している
- 日本語テキストの行高さは1.5-1.8が標準

### 3. 複数ニュースカテゴリの取得：Python スクリプト拡張
**決定**: `fetch_news.py` を拡張し、複数のNHKニュースRSSフィード（`cat0`=主要、`cat1`=地域別等）を並列で取得

**実装方針**:
- `fetch_news.py` に複数RSSフィード定義を追加
- 各フィードから指定件数（主要10件、カテゴリ別5件）を取得
- 取得データを `data/news.json` に「main_news」「category_news」キーで構造化

**根拠**:
- NHKニュースが複数RSSフィード（`cat0`, `cat1`, `cat2`等）を提供している
- Pythonの `concurrent.futures` や非同期ライブラリで並列処理が実現可能
- JSONスキーマの拡張で既存の parser ロジックに大きな影響を与えない

### 4. データ構造の設計：JSON形式の新規スキーマ
**決定**: `news.json` を以下の構造に拡張（BREAKING CHANGE）
```json
{
  "main_news": [
    { "id": "...", "title": "...", "summary": "...", "content": "...", "translation": "...", "timestamp": "..." },
    ...
  ],
  "category_news": {
    "region_1": [ { ... }, ... ],
    "region_2": [ { ... }, ... ],
    ...
  }
}
```

**根拠**:
- 明示的なカテゴリ構造でフロントエンド側での処理が簡潔になる
- スケーラビリティ（新規カテゴリの追加が容易）
- フロントエンドで「主要ニュース」と「カテゴリ別ニュース」を別に表示できる

## Risks / Trade-offs

| リスク | 影響 | 対策 |
|--------|------|------|
| **データ構造の変更（BREAKING）** | 既存フロントエンド・ツールが破壊される可能性 | マイグレーションガイドを提供し、フロントエンド側でのデータアクセスをブラケット記法で明示化 |
| **複数RSSフィード取得時のネットワーク遅延** | GitHub Actions ワークフロー実行時間が増加 | タイムアウト設定を調整し、フィード取得失敗時のフォールバック機構を実装 |
| **UIテーマ色の認識差異** | 異なるブラウザ・デバイスでの色表現が異なる可能性 | ブラウザ互換性テストを実施、標準的なWeb色定義（HEX/RGB）を使用 |
| **記事タイトル表示の日本語折り返し不確実性** | 一部ブラウザで `word-break` の動作が異なる可能性 | CSS フォールバック（`overflow-wrap`, `white-space` との組み合わせ）で対応 |

## Migration Plan

1. **フェーズ1**: `style.css` を修正し、ホワイト基調のカラースキーム、タイトル表示CSSを反映
2. **フェーズ2**: `fetch_news.py` を拡張し、複数RSSフィード取得と新規JSON構造での出力を実装・テスト
3. **フェーズ3**: `index.html` と `app.js` を修正し、新規JSON構造に対応した表示ロジックに変更
4. **フェーズ4**: GitHub Actions ワークフロー（`.github/workflows/` 配下）で新規スクリプトがキックされるよう設定確認
5. **ロールバック**: 既存の `news.json` 形式を `news_legacy.json` として保存し、緊急時はそちらに切り替え
